# ПОЛЯНА — Game Design Specification

**Версия:** 0.1 (Draft)  
**Дата:** 2026-04-24  
**Репозиторий:** https://github.com/antioz/polana

---

## 1. Концепция

**ПОЛЯНА** — autochess-игра о футбольных хулиганах. Игроки собирают отряды бойцов из фанатских группировок («фирм») и выставляют их на автоматический бой на поляне в городском парке.

Жанровая основа: **Dota Underlords** — фазы покупки/расстановки чередуются с автоматическими боями.

**Целевая аудитория:** фанаты фанатской субкультуры, игроки autochess, PC.

**Цель продукта:** коммерческий релиз. Первый этап — демо с ботами.

---

## 2. Сеттинг

### Поле боя
Поляна в городском парке: трава, деревья по краям, естественное освещение. Вид сверху (top-down, как TFT).

### Фоновые персонажи (ambient, не играбельные)
Создают атмосферу и узнаваемость сцены:

| Персонаж | Роль |
|---|---|
| Дед | Случайный прохожий, реагирует на драку |
| Мама с коляской | Спешно уходит |
| Велосипедист | Тормозит, смотрит |
| 2 оператора с камерами | **Обязательно**, с разных сторон поляны |
| Тренеры | Стоят по краям, жестикулируют |
| Смотрящие | Молча наблюдают, не вмешиваются |

---

## 3. Игровой цикл

```
[Выбор фракции] → [Фаза покупки] → [Расстановка] → [Бой] → [Результат] → [Фаза покупки] → ...
```

### Фазы раунда

1. **Выбор фракции** (один раз перед матчем) — игрок выбирает «фирму», получает пассивный бонус на весь матч
2. **Фаза покупки** — в магазине появляются случайные юниты, покупаются за золото
3. **Расстановка** — игрок расставляет юнитов на своей половине поля (сетка hex или квадратная — TBD)
4. **Бой** — юниты дерутся автоматически по правилам AI
5. **Результат** — проигравший теряет HP, выигравший получает золото/бонусы
6. **Повтор** до победы (один выживший) или конца демо-сценария

---

## 4. Система фракций

Фракция выбирается **один раз до матча**. Это «фирма» — вымышленная фанатская группировка.

> ⚠️ Конкретные названия фирм и баланс бонусов — TBD (итерация 2)

**Принцип баланса:** каждая фракция усиливает определённый архетип юнитов. Примеры (placeholder):

| Фракция | Бонус |
|---|---|
| Северные | +15% урон Спортиков |
| Южный берег | +20% HP Отморозков |
| Выезд 007 | Выездные дают ауру на 1 клетку дальше |

---

## 5. Архетипы юнитов

Три архетипа — основа системы. Каждый юнит принадлежит одному архетипу.

### Подписной спортик
**Роль:** Технарь, контроль, честный бой  
**Атака:** Single-target (один противник)  
**Особенности:**
- Работает лучше в позиции 1v1
- Способность: оглушение / выбивание из позиции
- Слабее если окружён (нет AoE)

### Отморозок
**Роль:** Хаос, берсерк, высокий риск/награда  
**Атака:** AoE (по площади, несколько врагов)  
**Особенности:**
- Может случайно ударить союзника
- Способность: впадает в ярость — резко растёт урон, падает защита
- Непредсказуем: поведение частично рандомное

### Выездной
**Роль:** Опыт, поддержка, командная игра  
**Атака:** Buff/aura (усиление союзников)  
**Особенности:**
- В одиночку слабый
- В группе 2+ Выездных: аура даёт бонус морали всей команде
- Способность: «видел виды» — иммунитет к первому оглушению

## 6. Классы юнитов

Каждый класс — конкретный персонаж внутри архетипа. Разные статы, разная способность.

### Подписной спортик

| Класс | Особенность |
|---|---|
| МСМК по боксу | Высокий урон, точные удары, оглушение |
| Чемпион Беллатура | Захват + бросок (меняет позицию противника) |
| Двоюродный брат из спецназа | Высокая защита, контратака |
| Знакомый боксерчик | Дешёвый, слабый, но доступен с 1 раунда |

### Отморозок

| Класс | Особенность |
|---|---|
| Сидевший | Игнорирует болевой порог — не уходит в стан |
| Бешеный торч | Максимальный хаос, атакует всё подряд |
| Детдомовец | Берсерк от низкого HP — урон растёт |
| БС | Самый дорогой, самый непредсказуемый |

### Выездной

| Класс | Особенность |
|---|---|
| Боевой ультрас | Сильнейший в архетипе, аура дальше |
| Пьянь | Дешёвый, аура слабее, но занимает слот фракции |

> ⚠️ Архетипы неполные — расширить в итерации 2

## 7. Визуал и ассеты

**Стиль:** 3D, реалистичный (спортивные костюмы, узнаваемая эстетика забива)  
**Камера:** Top-down, угол ~60° (как TFT)  
**Освещение:** Дневное, естественное

### Asset Pipeline (демо)

```
Концепт/референс → Meshy.ai → 3D модель (.fbx/.glb)
                                        ↓
                              Импорт в Unity
                                        ↓
                    Mixamo (Adobe) → Анимации (удар/падение/бег/уклонение)
                                        ↓
                          Текстуры под цвет фракции
```

**Референсы:** видео и фото реальных забивов (предоставлены владельцем проекта).

## 9. Подход к разработке

### Агент-ориентированная разработка
Проект разрабатывается через Claude Code с Unity MCP:
- Агенты пишут C# скрипты и управляют Unity Editor через MCP
- Каждый блок работы — отдельная зона ответственности агента
- Координация через Claude Code в терминале Cursor

### Зоны ответственности агентов

| Агент | Зона |
|---|---|
| Architecture agent | Структура проекта, базовые системы |
| Gameplay agent | Механика боя, AI юнитов |
| UI agent | Интерфейс магазина, расстановки, результатов |
| Assets agent | Импорт моделей, настройка анимаций |
| Balance agent | Статы юнитов, баланс фракций |
